# 접근 제어자 (Access Modifiers)

## Java 접근 제어자

Java는 4가지 접근 제어자를 제공합니다.

| 접근 제어자                      | 같은 클래스 | 같은 패키지 | 자식 클래스 | 외부 패키지 |
|-----------------------------|--------|--------|--------|--------|
| `public`                    | ✅      | ✅      | ✅      | ✅      |
| `protected`                 | ✅      | ✅      | ✅      | ❌      |
| `default` (package-private) | ✅      | ✅      | ❌      | ❌      |
| `private`                   | ✅      | ❌      | ❌      | ❌      |

### 1. public

```java
public class MyClass {
    public int value;

    public void method() {
    }
}
```

- 모든 곳에서 접근 가능
- 클래스, 메서드, 필드에 모두 사용 가능

### 2. protected

```java
public class Parent {
    protected int value;

    protected void method() {
    }
}
```

- 같은 패키지 내에서 접근 가능
- 다른 패키지의 자식 클래스에서 접근 가능

### 3. default (package-private)

```java
class MyClass {  // 접근 제어자 생략
    int value;

    void method() {
    }
}
```

- **접근 제어자를 명시하지 않으면 기본값**
- 같은 패키지 내에서만 접근 가능
- 다른 패키지에서는 접근 불가

### 4. private

```java
public class MyClass {
    private int value;

    private void method() {
    }
}
```

- 같은 클래스 내에서만 접근 가능
- 가장 제한적인 접근 제어자

## Kotlin 접근 제어자

Kotlin은 4가지 접근 제어자를 제공하며, Java와 다른 점이 있습니다.

| 접근 제어자      | 같은 파일 | 같은 모듈 | 자식 클래스 | 외부 모듈 |
|-------------|-------|-------|--------|-------|
| `public`    | ✅     | ✅     | ✅      | ✅     |
| `internal`  | ✅     | ✅     | ✅      | ❌     |
| `protected` | ✅     | -     | ✅      | ❌     |
| `private`   | ✅     | ❌     | ❌      | ❌     |

### 1. public (기본값)

```kotlin
class MyClass {  // public이 기본값
    val value: Int = 0
    fun method() {}
}

public class ExplicitPublic {
    public val value: Int = 0
}
```

- **Kotlin에서는 public이 기본값** (Java와 반대)
- 모든 곳에서 접근 가능

### 2. internal

```kotlin
internal class MyClass {
    internal val value: Int = 0
    internal fun method() {}
}
```

- **Kotlin에만 있는 접근 제어자**
- 같은 모듈(module) 내에서만 접근 가능
- 모듈: Gradle, Maven 프로젝트 단위

### 3. protected

```kotlin
open class Parent {
    protected val value: Int = 0
    protected fun method() {}
}

class Child : Parent() {
    fun test() {
        println(value)  // OK: 자식 클래스에서 접근 가능
    }
}
```

- 클래스 멤버에만 사용 가능 (최상위 선언에는 사용 불가)
- 자식 클래스에서 접근 가능
- **Java와 달리 같은 패키지라도 자식 클래스가 아니면 접근 불가**

### 4. private

```kotlin
private class MyClass {
    private val value: Int = 0
    private fun method() {}
}
```

- 같은 파일 또는 같은 클래스 내에서만 접근 가능
- 최상위 선언에 사용하면 같은 파일 내에서만 접근 가능

## 주요 차이점 비교

### 1. 기본 접근 제어자

| 언어         | 기본값                         |
|------------|-----------------------------|
| **Java**   | `default` (package-private) |
| **Kotlin** | `public`                    |

```java
// Java: package-private
class MyClass {
}
```

```kotlin
// Kotlin: public
class MyClass {}
```

### 2. package-private vs internal

- **Java**: `default` (package-private) - 같은 패키지 내에서 접근 가능
- **Kotlin**: `internal` - 같은 모듈 내에서 접근 가능

Kotlin에는 Java의 package-private에 해당하는 접근 제어자가 없습니다.

### 3. protected 차이

- **Java**: 같은 패키지 + 자식 클래스
- **Kotlin**: 자식 클래스만 (같은 패키지여도 자식 클래스가 아니면 접근 불가)

### 4. 최상위 선언

```kotlin
// Kotlin에서 최상위 선언 가능
private fun topLevelFunction() {}  // 같은 파일에서만 접근
internal val topLevelValue = 10     // 같은 모듈에서만 접근
```

Java에서는 최상위 클래스만 public 또는 default 가능.

## 실전 예제: Java와 Kotlin 상호 운용

### 문제 상황

```java
// Java
package src;

class Fun11720 {  // default (package-private)
    public void solve() {
    }
}
```

```kotlin
// Kotlin
package src.main.kotlin

import src.Fun11720

fun main() {
    valfun = Fun11720()  // ❌ 에러: Cannot access
}
```

### 해결 방법

```java
// Java: public으로 변경
package src;

public class Fun11720 {  // public으로 변경
    public void solve() {
    }
}
```

```kotlin
// Kotlin
package src.main.kotlin

import src.Fun11720

fun main() {
    valfun = Fun11720()  // ✅ 정상 동작
    fun .solve()
}
```

## 권장 사항

### Java

1. API로 공개할 클래스/메서드는 `public` 사용
2. 상속을 고려한 멤버는 `protected` 사용
3. 패키지 내부에서만 사용할 클래스는 `default` 유지
4. 클래스 내부 구현은 `private` 사용

### Kotlin

1. 기본적으로 `public`이므로 명시 불필요
2. 모듈 내부 API는 `internal` 사용
3. 상속을 위한 멤버는 `protected` 사용
4. 구현 세부사항은 `private` 사용

### Java-Kotlin 혼용 프로젝트

- Java 클래스를 Kotlin에서 사용하려면 `public` 명시 필요
- Kotlin의 `internal`은 Java에서 `public`으로 보임 (주의!)
- 명확한 접근 제어자 사용 권장
